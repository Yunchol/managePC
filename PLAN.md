# 学童ゲーム時間管理システム 実装計画

## システム構成
- **スタッフPC**: Go サーバー + Web UI をサーブ
- **子どもPC × 6**: Go エージェント（常駐 exe）
- **スタッフのスマホ**: ブラウザで Web UI にアクセス

---

## 実装ステップ

### Step 1: プロジェクト構成を作る
**達成したいこと**: ディレクトリ構成を整えて Go モジュールを初期化する

- [ ] ディレクトリ構成を作る（server/ agent/ ui/）
- [ ] `go mod init` でモジュール初期化（server・agent それぞれ）
- [ ] 依存パッケージ追加（gorilla/websocket）

---

### Step 2: Go サーバー — 基礎
**達成したいこと**: サーバーが起動して WebSocket 接続を受け付けられる

- [ ] HTTP サーバーを起動する（ポート 8080）
- [ ] WebSocket エンドポイントを作る（`/ws`）
- [ ] エージェントが接続してきたら PC 名を登録して管理する
- [ ] 接続中の PC 一覧をログに出す

---

### Step 3: Go エージェント — 基礎
**達成したいこと**: 子どもPC 側のエージェントがサーバーに接続して名乗れる

- [ ] WebSocket でサーバーに接続する
- [ ] 起動時に PC 名（例: `pc-1`）をサーバーに送る
- [ ] サーバーとの接続が切れたら自動再接続する
- [ ] Windows で単体 exe としてビルドできる（クロスコンパイル）

---

### Step 4: タイマー機能
**達成したいこと**: サーバーからタイマーを送って、エージェントがカウントダウンできる

- [ ] サーバー → エージェントへ「◯分タイマー開始」メッセージを送る
- [ ] エージェントがカウントダウンを開始する
- [ ] 残り時間をサーバーに定期報告する（スタッフ UI に表示するため）
- [ ] 残り 5 分・1 分で警告（ダイアログ表示）する

---

### Step 5: ゲーム終了機能
**達成したいこと**: 時間になったらゲームのプロセスを kill できる

- [ ] タイマー終了時にゲームプロセスを kill する
- [ ] Chrome（ブラウザゲーム）を終了する
- [ ] マイクラ Java 版（`javaw.exe`）を終了する
- [ ] マイクラ統合版（`Minecraft.Windows.exe`）を終了する
- [ ] 終了完了をサーバーに報告する

---

### Step 6: Web UI — スタッフ管理画面
**達成したいこと**: スマホから PC ごとにタイマーを設定・確認できる

- [ ] 接続中の PC 一覧を表示する
- [ ] PC ごとに「15分」「30分」「60分」ボタンを置く
- [ ] 残り時間をリアルタイムで表示する（WebSocket で更新）
- [ ] 終了したら「終了しました」と表示する
- [ ] スマホで見やすいレイアウトにする

---

### Step 7: 再起動検知
**達成したいこと**: 子どもが PC を再起動したらスタッフに通知する

- [ ] エージェント起動時に「再接続」か「初回接続」かを判定する
- [ ] タイマー中に再接続してきた場合、スタッフ UI に通知する
- [ ] 再接続後もタイマーを継続するか選べるようにする（任意）

---

### Step 8: 自動起動設定
**達成したいこと**: PC 起動時に自動でサーバー・エージェントが立ち上がる

- [ ] エージェントを Windows サービスとして登録するスクリプトを作る
- [ ] サーバーをスタッフ PC のスタートアップに登録する
- [ ] 動作確認（再起動後も自動で繋がるか）

---

### Step 9: 動作テスト・調整
**達成したいこと**: 実際の学童環境で問題なく動くことを確認する

- [ ] 6 台同時接続のテスト
- [ ] スタッフのスマホから操作できるか確認
- [ ] ゲーム終了が確実に動くか確認
- [ ] ネットワークが不安定な状況でも再接続できるか確認

---

## 現在の進捗
**Step 1 から開始**

## メモ
- サーバー: スタッフPC（LAN 内固定 IP）
- エージェント: クロスコンパイル（Mac で Windows 用 exe を生成）
- スタッフアクセス: `http://192.168.x.x:8080`（同一 Wi-Fi 必須）
- 子どもPC の IP 固定は不要（エージェント側から接続しに行くため）
